# Assistive Switch Collection

This repository is a curated collection of open‑source assistive **switches** and **switch interfaces**, collected as Git submodules and indexed in a machine‑readable CSV file.

The goal is to make it easier for clinicians, AT professionals, and makers to discover, compare, and build on existing open hardware.

---

## Repository structure

- `switch_projects.csv` — master index of projects (name, repo URL, type, number of inputs, IO mode, license, and whether 3D models are available).  
- `repos/` — Git submodules pointing at each upstream project.  
- `docs/` (optional) — any generated site or additional documentation.

---

## Project metadata fields

Each row in `switch_projects.csv` uses the following columns:

- `name` — human‑readable project name.  
- `github_repo` — canonical repository URL or project page if the code is linked from there.  
- `type` — one of:
  - `switch` for physical access switches (typically 3.5 mm).  
  - `interface` for devices that turn one or more switches (or sip‑and‑puff) into USB HID, gamepad, network, or relay outputs.  
  - `switch_controlled_device` for projects where the main goal is controlling something else (e.g., coffee maker, water pump) with an assistive switch.  
- `inputs` — approximate number and type of user inputs (e.g., `1`, `2`, `2+ sip_puff`).  
- `io_mode` — how the device presents itself or actuates:
  - `USB_HID`, `Bluetooth_HID`, `relay`, `network_to_servo`, `contact_closure`, etc.  
- `license` — SPDX identifier or short description (e.g., `MIT`, `CC-BY-NC-SA 4.0`).  
- `has_3d_files` — `yes` / `no` / `unknown`.  
- `notes` — brief free‑text description, special features, or caveats (e.g., non‑commercial license, requires specific microcontroller).
